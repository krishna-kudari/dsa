LC 1438 — Longest Subarray with Abs Diff ≤ K

Interactive visualizer · O(N) two-deque solution · Uber SDE1/SDE2 pattern

The core question: For every right pointer position r, what is the smallest left pointer L such that max(arr[L..r]) − min(arr[L..r]) ≤ K?
Why naive fails

Brute force checks every pair (i,j) and scans the subarray for min/max → O(N³). The bottleneck is recomputing min/max from scratch each time the window moves.

Key insight: When we slide a window right by one, we only ADD one element on the right. The min/max can only change at the edges — this is what a monotonic deque exploits.

Two-pointer alone isn't enough. Unlike "sum ≤ K" problems, shrinking the window doesn't guarantee the condition restores — removing an element from the left might remove the current max OR min. You need O(1) access to both max and min at all times.
The deque insight

Maintain two deques simultaneously:
— max_dq: indices in decreasing value order → front = window max
— min_dq: indices in increasing value order → front = window min

When you add element at index r:
1. Pop max_dq back while arr[back] ≤ arr[r]
2. Pop min_dq back while arr[back] ≥ arr[r]
3. If arr[max_dq[0]] − arr[min_dq[0]] > K, advance left, evicting stale fronts
4. Window size = r − left + 1

Every element is added to each deque once and removed at most once → O(N) total. Window only ever moves right → two-pointer validity.

Why sorted set is worse than deque

A sorted set gives O(log N) insert/delete and O(1) min/max — O(N log N) overall. A monotonic deque achieves O(1) amortized per element by being lazily invalidated — you only keep elements that could possibly be min/max in future windows. The deque is always a subset of the window.

Python — O(N) two deque solution
from collections import deque

def longestSubarray(nums, limit):
    max_dq = deque()  # indices, values decreasing → front = max
    min_dq = deque()  # indices, values increasing → front = min
    left = 0
    best = 0

    for right in range(len(nums)):
        # maintain max_dq: remove back while back val ≤ current
        while max_dq and nums[max_dq[-1]] <= nums[right]:
            max_dq.pop()
        max_dq.append(right)

        # maintain min_dq: remove back while back val >= current
        while min_dq and nums[min_dq[-1]] >= nums[right]:
            min_dq.pop()
        min_dq.append(right)

        # shrink window if constraint violated
        while nums[max_dq[0]] - nums[min_dq[0]] > limit:
            left += 1
            if max_dq[0] < left:
                max_dq.popleft()
            if min_dq[0] < left:
                min_dq.popleft()

        best = max(best, right - left + 1)

    return best
Java — same logic
public int longestSubarray(int[] nums, int limit) {
    Deque<Integer> maxDq = new ArrayDeque<>();
    Deque<Integer> minDq = new ArrayDeque<>();
    int left = 0, best = 0;

    for (int right = 0; right < nums.length; right++) {
        while (!maxDq.isEmpty() && nums[maxDq.peekLast()] <= nums[right])
            maxDq.pollLast();
        maxDq.addLast(right);

        while (!minDq.isEmpty() && nums[minDq.peekLast()] >= nums[right])
            minDq.pollLast();
        minDq.addLast(right);

        while (nums[maxDq.peekFirst()] - nums[minDq.peekFirst()] > limit) {
            left++;
            if (maxDq.peekFirst() < left) maxDq.pollFirst();
            if (minDq.peekFirst() < left) minDq.pollFirst();
        }
        best = Math.max(best, right - left + 1);
    }
    return best;
}
Edge cases to mention
K = 0 → only subarrays of all identical elements. Works correctly — window shrinks until max == min.
All elements equal → entire array is valid. max_dq and min_dq fronts always equal → diff = 0 ≤ K always.
Single element → always valid, returns 1.
Duplicate values: deques store indices (not values), so duplicates are handled naturally.
What makes this answer strong

1. Restate the constraint: "I need max(window) − min(window) ≤ K at every step."
2. Name the bottleneck: "The hard part is maintaining max and min as the window slides."
3. Show the arc: O(N³) → O(N²) → O(N log N) → O(N), explaining each transition.
4. Prove O(N): "Each element appended once, removed at most once per deque — amortized O(1), O(N) total."
5. Prove correctness: "Left pointer only moves forward. Each time the constraint is violated, we must advance left — no smaller starting point fixes it."
